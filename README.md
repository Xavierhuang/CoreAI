# CoreAI — On-Device AI Comparison IDE

A SwiftUI macOS IDE built as a hands-on testbed for the question:
**"How much can on-device AI models actually do on a Mac in 2026?"**

Five swappable AI backends behind one chat panel — Claude, Apple
Intelligence (on-device), Apple Private Cloud Compute, Ollama
(local open-source), and Apple's Core AI runtime — all asking the
same questions over the same project folder so you can compare them
apples-to-apples.

Also includes the slide decks for the accompanying talk.

## What's in the repo

```
ide/                          SwiftUI app source
ideTests/                     XCTest target
ide.xcodeproj/                Xcode project
CodeIDE/                      original Obj-C version (reference)
build_core_ai_deck.py         6-slide lightning-talk generator
build_core_ai_deck_long.py    16-slide long-form generator
*.pptx                        generated decks
```

## The 5 backends

| Backend | Where it runs | Notes |
|---|---|---|
| **Claude** | Anthropic cloud via the `claude` CLI | Agentic, can edit files. Best quality. |
| **Apple (Local)** | `SystemLanguageModel` via FoundationModels | macOS 26+, ~3B on-device, has tools |
| **Apple (PCC)** | `PrivateCloudComputeLanguageModel` | macOS 27+, bigger Apple model in attested cloud |
| **Ollama** | `localhost:11434` daemon | Any GGUF model you `ollama pull`; supports tool calls |
| **Core AI** | `llm-runner` subprocess against `.aimodel` | macOS 27+ runtime; build the binary from [apple/coreai-models](https://github.com/apple/coreai-models) |

Each backend keeps its own conversation history so switching the
picker doesn't reset others. Same `Tool` protocol (`listFiles`,
`readFile`) where supported; pre-injected project file tree as a
fallback where not.

## Requirements

- macOS 14.0+ to **run** the basic app
- macOS 26+ for Apple Intelligence backend
- macOS 27+ for Apple PCC and Core AI backends
- Xcode 26+ (the SDK has FoundationModels, weak-linked via `@available`)
- Optional: [Claude Code](https://claude.com/code) CLI for the Claude backend
- Optional: [Ollama](https://ollama.com) for the local open-source backend
- Optional: a `.aimodel` exported via `apple/coreai-models` for the Core AI backend

## Build

```sh

    -configuration Debug -destination 'platform=macOS' build

# Or just open in Xcode and hit Run.
open ide.xcodeproj
```

## The talk

Two versions of the same talk, generated by Python scripts:

- **`On-Device-AI-Mac-Talk.pptx`** — 6 slides for a 6-minute lightning talk
- **`On-Device-AI-Mac-Talk-Long.pptx`** — 16 slides for a 20–30 min slot

Regenerate either after editing the source script:

```sh
python3 -m venv /tmp/pptx-venv
/tmp/pptx-venv/bin/pip install python-pptx
/tmp/pptx-venv/bin/python build_core_ai_deck.py        # 6-slide
/tmp/pptx-venv/bin/python build_core_ai_deck_long.py   # 16-slide
```

Speaker notes are baked into every slide — open Keynote's presenter
view (⌥⌘P) to see them alongside.

## Notable bits inside the code

- Project context is set ONCE in `LanguageModelSession.instructions`,
  not per-turn — otherwise the rolling transcript trips the small
  on-device model's context window after a few turns.
- Ollama's `/api/chat` is called with the `tools` array; if a model
  rejects with `HTTP 400 "does not support tools"`, we transparently
  retry without tools (handles older models like `deepseek-coder:6.7b`).
- `Send` button toggles to `Stop` while a turn is in flight;
  cancellation propagates into `URLSession` and `LanguageModelSession`
  natively. `Process`-based backends (Claude CLI, Core AI) leave the
  child running but the UI moves on.
- Live free-RAM badge (`SystemStats.swift` uses Mach `host_statistics64`)
  so you can see at a glance whether the next Ollama turn will land on
  Metal or fall back to CPU.
